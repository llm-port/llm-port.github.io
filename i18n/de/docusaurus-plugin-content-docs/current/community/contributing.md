# Contributing

Contributions are welcome.

## How to contribute

1. Fork the relevant repository
2. Create a feature branch from `main`
3. Implement and test your changes
4. Open a pull request with a clear summary

## Contribution expectations

- Follow repository coding and review standards
- Include tests or validation evidence for behavior changes
- Keep pull requests focused and reviewable

## Licensing and agreements

Contributions are licensed under the project’s published license terms.
If your workflow requires a contributor agreement, complete it before merge.

## Support channels

Use project issues and discussions for questions, bug reports, and proposals.
