# PII 检测与脱敏

**PII 模块**（`llm_port_pii`）提供基于 Presidio 的 PII 扫描、脱敏和可逆令牌化，集成到 Gateway 管道中。

## 工作原理

PII 检测作为独立的**无状态** FastAPI 服务运行，Gateway 在处理请求时调用它。启用后，出站请求在到达上游提供商之前可被扫描个人身份信息。扫描事件通过 HTTP 转发到 Backend 进行持久化存储。

## 实体类型

可配置的检测实体：

| 实体            | 示例                          |
| --------------- | ----------------------------- |
| `PERSON`        | 张三                          |
| `EMAIL_ADDRESS` | user@example.com              |
| `PHONE_NUMBER`  | +86-555-0100                  |
| `CREDIT_CARD`   | 4111-1111-1111-1111           |
| `US_SSN`        | 123-45-6789                   |
| `IP_ADDRESS`    | 192.168.1.1                   |
| `LOCATION`      | 北京市                        |
| _自定义实体_    | 通过 Presidio recognizer 配置 |

## 策略

PII 策略遵循**优先级层次**：

1. **租户策略** — 每个租户的覆盖配置
2. **系统默认** — 全局基准策略
3. **无** — PII 扫描禁用

每个策略配置：

- 检测哪些实体类型
- 脱敏策略（遮掩、替换、哈希）
- 失败模式行为

## 失败模式

| 模式       | 行为                     |
| ---------- | ------------------------ |
| `block`    | PII 服务不可达时拒绝请求 |
| `allow`    | PII 服务不可达时放行请求 |
| `fallback` | 回退到本地基于规则的检测 |

## Gateway 集成

- **遥测净化**：根据隐私模式从 Langfuse 追踪中脱敏 PII 实体
- **出站控制**：出站请求在到达远程提供商之前可被拦截或脱敏
- **模块感知 UI**：模块禁用时，管理控制台隐藏 PII 设置

## 仪表板与事件日志

管理控制台包含 **PII 仪表板**，用于对已识别的实体、脱敏操作和扫描量进行可视化分析。

- **事件日志**：每次扫描事件（实体计数、脱敏方式、时间戳）均记录到 backend 数据库，可浏览、导出或用于合规审计
- **租户策略管理**：管理员可通过 UI 为每个租户创建、编辑和应用 PII 策略
