# RAG（检索增强生成）

**RAG 模块**（`llm_port_rag`）提供完整的文档摄入和检索管道。

## 知识搜索

三种自动评分的搜索策略：

| 策略       | 工作原理                               |
| ---------- | -------------------------------------- |
| **向量**   | pgvector 嵌入上的余弦相似度            |
| **关键词** | 带 ts_rank 评分的全文搜索              |
| **混合**   | 向量 + 关键词的加权组合，使用 RRF 融合 |

所有搜索均强制执行：

- **租户隔离** — 按租户 + 可选工作区分区
- **ACL 执行** — 块级访问控制主体
- **过滤支持** — 容器、资产和自定义字段的元数据过滤

## 文档管道

摄入管道通过一系列阶段处理上传：

```
上传 → MinIO → 提取 → 归一化 → 分块 → 嵌入 → 索引 (pgvector)
```

### 上传

- **预签名 URL** — 通过预签名 PUT 浏览器直接上传到 MinIO
- **SHA-256 去重** — 重新发布时跳过未更改的文件

### 提取与分块

- 可配置的块大小和重叠
- 从控制平面推送的运行时嵌入配置
- 支持主流文档格式（PDF、DOCX、TXT、HTML、Markdown）

### 嵌入

- 每个部署可配置提供商和模型
- 嵌入请求通过 Gateway 自己的提供商池路由
- 批量嵌入提升效率

## 虚拟容器

用于组织知识的 N 层容器树：

- **容器**存储资产（文档）
- **资产**具有带草稿/发布工作流的版本
- **草稿 → 发布**模型，支持可选计划发布

## 采集器插件

用于自动内容同步的可插拔连接器：

| 连接器           | 状态                     |
| ---------------- | ------------------------ |
| 本地文件夹 / SMB | 可用                     |
| SharePoint       | Stub（可扩展）           |
| _自定义_         | 自定义连接器的插件注册表 |

采集器通过 Taskiq + RabbitMQ 按可配置计划运行。

## 异步处理

- **Taskiq** 任务运行器，以 RabbitMQ 为消息代理
- 用于摄入、发布和计划操作的后台 Worker
- 带状态事件和进度报告的作业追踪
