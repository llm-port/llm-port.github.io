# Compliance and Third-Party Notices

llm.port depends on third-party components and libraries. Each component is governed by its own license terms.

## Public commitments

- Maintain clear attribution for third-party dependencies
- Track license obligations for distributed components
- Provide updates when dependency posture changes materially

## Recommended compliance operations

- Keep a current third-party inventory (SBOM)
- Pin production dependency versions where possible
- Include license review in release governance

## Scope

This public page provides policy-level clarity. Detailed inventory records and component-level legal tracking should remain in internal compliance documentation.
