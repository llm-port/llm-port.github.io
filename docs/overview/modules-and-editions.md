# Modules and Editions

llm.port uses a modular model so teams can start lean and expand over time.

## Core capabilities

Core platform capabilities include:

- OpenAI-compatible API Gateway
- Central administration and policy controls
- Base chat/session functionality

## Optional modules

Common optional modules include:

- **PII Controls** for scanning and redaction workflows
- **RAG** for document ingestion and retrieval workflows
- **MCP Tools** for governed external tool usage
- **Node Agent** for multi-node operations

## Edition model

Some modules may be available only in specific editions. Use your deployment documentation and licensing terms as the final source for entitlement.

## Module lifecycle

You can enable modules as requirements evolve, without changing the core product adoption flow.

## Guidance for public docs

Public pages should explain:

- What each module enables
- When to enable it
- High-level operational impact

Public pages should avoid exposing internal service wiring or implementation internals.
